## 延迟分析

**平均延迟: 10964ms (约 11 秒)**

### 延迟分解

1. **Embedding 生成:** ~500-1000ms
2. **向量检索 (Qdrant):** ~100-300ms
3. **LLM 重排序:** ~9000-10000ms (主要瓶颈)

### 优化建议

1. **引入专用 Reranker 模型** - 如 BGE-Reranker，延迟可降至 < 200ms
2. **缓存热点查询结果** - 减少 Embedding 计算
3. **并行重排序** - 批量处理多个候选

   - 建议：评估专用 Reranker 模型替代方案
