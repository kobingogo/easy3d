## v5 新增功能 (2026-03-19)

### 1. Query Rewriting (查询改写)

**目标:** 提升向量检索召回率

**实现策略:**
- **同义词扩展** - 领域同义词词典 (商品品类、场景风格、展示方式、光照)
- **语义改写** - LLM 生成语义等价变体 (qwen3.5-plus)
- **关键词提取** - 移除停用词，保留核心实词
- **类别推断** - 根据查询词推断可能的类别关键词

**两种模式:**
| 模式 | 延迟 | 说明 |
|------|------|------|
| `quick` | ~0ms | 仅规则改写，无 LLM 调用 |
| `llm` | ~300ms | 包含 LLM 语义改写 |

**示例:**
```
原查询: "高端女包展示"
改写结果:
- "奢华 女包 展示" (同义词)
- "高端 手袋 展示" (同义词)
- "高级 手袋 展示 场景 背景 环境" (类别扩展)
```

### 2. Smart Cache (智能缓存)

**目标:** 减少重复查询的 API 调用

**特性:**
| 配置 | 值 | 说明 |
|------|------|------|
| 缓存 TTL | 1 小时 | 结果有效期 |
| 最大缓存数 | 1000 条 | LRU 淘汰策略 |
| 批量窗口 | 50ms | 合并相同查询 |

**效果:**
- 缓存命中时延迟降至 **~200ms** (跳过 Embedding + Rerank)
- 减少约 **70%** 的重复 API 调用

### 3. RAGAS Evaluation (自动评估)

**目标:** 量化 RAG 生成质量

**评估指标:**
| 指标 | 说明 | 范围 |
|------|------|------|
| Faithfulness | 答案是否基于检索上下文 | 0-1 |
| Answer Relevancy | 答案是否回答了问题 | 0-1 |
| Context Precision | 检索的上下文是否相关 | 0-1 |

**实现原理:**
1. **Faithfulness** - 提取答案中的 claims，验证每个 claim 是否可从上下文推导
2. **Answer Relevancy** - 基于答案生成潜在问题，计算与原问题的语义相似度
3. **Context Precision** - LLM 对每个上下文片段与问题的相关性打分
