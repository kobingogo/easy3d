## 1. 产品愿景与定位

### 1.1 产品愿景

**让每个电商卖家都能 30 秒生成专业级产品 3D 展示**

### 1.2 核心价值主张

| 维度 | 传统方案 | easy3d 方案 | 提升幅度 |
|------|----------|-------------|----------|
| **成本** | 专业摄影 3000 元/天 | AI 生成 30 元/个 | **-99%** |
| **速度** | 3 天（拍摄+修图） | 30 秒 | **-99.9%** |
| **质量** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ (80%) | 专业级 80% |

### 1.3 目标用户

**核心用户画像**:

| 用户类型 | 年营收 | 团队规模 | 核心痛点 | 关键需求 |
|----------|--------|----------|----------|----------|
| 女装卖家 | 100-500 万 | 2-5 人 | 拍摄成本高、自己拍丑 | 便宜、快速、效果好 |
| 3C 数码卖家 | 500-2000 万 | 10-30 人 | 批量效率低、多平台适配 | 批量生成、多尺寸输出 |
| 美妆博主 | 50-200 万 | 1-3 人 | 文案没灵感、数据差 | 一键发布、种草文案 |

### 1.4 产品定位

从"AI 前端能力展示作品"演化为 **"电商卖家的 3D 展示生产力工具"**。

## 2. 三大核心能力

### 能力 1: 材质语义对齐 (技术壁垒)

**用户价值**: 用户说"丝滑"，系统懂 `Roughness: 0.2, Metalness: 0.1`

**实现方案**:
- 知识库规模: 508 条材质/场景/灯光/风格知识 ✅
- 检索精度: Precision@5 = 88.04% ✅
- 召回率: Recall@10 = 92.39% ✅

**验收标准**:
| 指标 | 目标值 | 实际值 | 状态 |
|------|--------|--------|------|
| 材质覆盖度 | 50+ 条目 | 508 条 | ✅ |
| 检索准确率 | >60% | 88.04% | ✅ |
| 召回率 | >90% | 92.39% | ✅ |

### 能力 2: 一键发布小红书

**用户价值**: 生成 → 适配 → 文案 → 发布，全流程自动化

**实现状态**:
- [ ] 多平台尺寸适配（淘宝 800x800、小红书 3:4、抖音 9:16）⏳ 待开发
- [ ] 小红书 OAuth 2.0 授权
- [ ] 自动生成种草文案
- [ ] 调用小红书 API 发布

### 能力 3: 电商场景优化

**用户价值**: 懂平台规范、懂电商语言、懂卖家需求

**平台规范知识库**:
| 平台 | 尺寸规格 | 格式要求 | 特殊要求 |
|------|----------|----------|----------|
| 淘宝主图 | 800x800 | JPG/PNG, ≤3MB | 白底、产品占比>70% |
| 小红书 | 1242x1660 (3:4) | JPG/PNG, ≤10MB | 竖图优先、可加文字 |
| 抖音 | 1080x1920 (9:16) | JPG/PNG, ≤5MB | 竖图、前3秒吸引眼球 |

## 3. 用户场景地图

### 场景 1: 女装卖家上新

**用户故事**:
```

easy3d流程: 上传照片 → 输入描述 → 生成3D → 一键输出 = 5分钟

时间节省: 91%

easy3d流程: 批量上传 → 选择模板 → 批量生成 → 打包下载 = 30分钟

时间节省: 99.6%

**需要的能力**:
- [ ] 批量上传

**用户故事**:
```
张小姐，26岁，小红书美妆博主
周末产出3篇种草笔记

当前流程: 试色拍照(2小时) + 修图(1小时) + 写文案(2小时) = 5小时/篇
easy3d流程: 上传产品照 → 选择模板 → 生成3D+文案 → 发布 = 30分钟/篇

时间节省: 90%

**需要的能力**:

## 4. 关键指标 (KPIs)

### 4.1 RAG 检索指标（已达成 ✅）

| 指标 | 目标值 | 实际值 | 状态 |
|------|--------|--------|------|
| Precision@5 | >60% | **88.04%** | ✅ +47% |
| Recall@10 | >90% | **92.39%** | ✅ +2.6% |
| Success Rate | >90% | **100%** | ✅ +11% |
| MRR | >0.7 | **0.7854** | ✅ +12% |
| 检索延迟 | <500ms | **724ms** | ⚠️ 待优化 |

### 4.2 Agent 工作流指标

| 指标 | 目标值 | 测量方法 |
|------|--------|----------|
| 生成成功率 | >90% | 成功生成/总请求 |
| 端到端延迟 | <60s | 用户输入 → 模型生成完成 |
| 工具调用准确率 | >95% | 参数正确 + 结果符合预期 |

### 4.3 商业化指标

| 指标 | 第1月 | 第3月 | 第6月 | 第12月 |
|------|-------|-------|-------|--------|
| 付费用户 | 30 | 100 | 500 | 2000 |
| MRR | 3000元 | 1万 | 5万 | 20万 |
| 毛利率 | 90% | 92% | 93% | 95% |

### 4.4 用户满意度指标

| 指标 | 目标值 | 测量方式 |
|------|--------|----------|
| NPS (净推荐值) | >30 | 用户满意度问卷 |
| 用户留存率 (D7) | >25% | 7日后回访用户/首日用户 |
| 生成→导出转化率 | >30% | 导出按钮点击/生成成功次数 |

## 8. 成本结构

### 固定成本

| 项目 | 月成本 |
|------|--------|
| 服务器（Vercel + Supabase） | 500 元 |
| Qdrant 向量库 | 300 元 |
| 域名 + SSL | 50 元 |
| **小计** | **850 元** |

### 变动成本

| 项目 | 单次成本 |
|------|----------|
| Tripo API | 0.35 元 |
| 百炼 LLM | 0.1 元 |
| 存储 + CDN | 0.05 元 |
| **小计** | **0.5 元** |

### 盈利模型

```
收入：10元/单 × 1000单 = 10000元/月
成本：固定850元 + 变动500元 + 营销2000元 = 3350元
利润：6650元/月
毛利率：93.5%
```
