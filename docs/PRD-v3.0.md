# easy3d 长期演进产品需求文档 (PRD)

## 1. 项目愿景与定位

easy3d 旨在从一个“AI 前端能力展示作品”演化为**“工业级、全流程 AI 3D 电商素材生产力工具”**。通过 RAG 知识库、自动化 Agent 集群和模型微调技术，解决 3D 生成中“不可控、不专业、难闭环”的痛点。

---

## 2. 演进路线图 (Evolution Roadmap)

### 第一阶段：体验与深度优化 (Short-term)

**目标**：提升现有功能的“确定性”与“透明度”。

#### 2.1 强化检索：混合检索 (Hybrid Search)

- **需求**：解决全向量检索在处理特定产品型号（如 "iPhone 16"）时的召回偏差。
- **功能**：引入 BM25 关键词索引，使用 RRF (Reciprocal Rank Fusion) 算法融合结果。

#### 2.2 增强信任：Agent 交互可视化

- **需求**：打破 Agent 的“黑盒”执行感。
- **功能**：在前端展示 Agent 的思考链路（Thought/Action/Observation），标注引用的知识库原文。

#### 2.3 效果保障：RAG 评估闭环

- **需求**：量化知识库效能。
- **功能**：接入 RAGAS 评估框架，建立 Faithfulness（忠实度）和 Relevancy（相关度）自动打分机制。

---

### 第二阶段：技术壁垒建设 (Mid-term)

**目标**：通过深度 AI 工程化建立项目差异化优势。

#### 2.4 专业化提示词：LoRA 模型微调

- **需求**：摆脱对通用大模型生成能力的依赖，使其更懂电商摄影语言。
- **功能**：基于现有的 500+ 专业样本，微调针对电商 3D 提示词优化的专属 LoRA 模型。

#### 2.5 性能与成本平衡：混合部署模式

- **需求**：降低对昂贵 API 的依赖，加速简单任务响应。
- **功能**：并在 Edge 或本地环境运行轻量级视觉识别模型（如分类、打标），复杂的生成任务保留在云端。

#### 2.6 复杂任务协同：多 Agent 协作系统

- **需求**：应对更复杂的场景设计。
- **功能**：拆分单一 Agent 为：场景专家 (Scene Designer)、光照专家 (Lighting Expert) 和合规合建员 (QC Inspector)。
- **协作协议**：
  - **Proposal**：场景专家提出初始方案（场景、灯光、材质参数）。
  - **Review**：材质专家审核参数合理性，QC Inspector 检查平台合规性。
  - **Execute**：通过后执行生成，否则退回修改（最多 2 轮重试）。
- **黑板架构**：各 Agent 读写共享状态，支持冲突解决（QC Agent 拥有最终否决权）。

---

### 第三阶段：商业化与生态集成 (Long-term)

**目标**：实现从“生成”到“发布”的商业全链路闭环。

#### 2.7 链路闭环：多平台自动化发布

- **需求**：生成的素材能够一键触达业务终端。
- **功能**：利用 OpenClaw 技能集成，一键同步至小红书、抖音、淘宝等电商平台。

#### 2.8 二次生产力：3D 在线轻量级编辑器

- **需求**：满足用户针对生成结果的微调需求。
- **功能**：提供基于浏览器端的材质换色、贴图替换、基础形变等编辑能力。

#### 2.9 行业赋能：SaaS 化行业模板

- **需求**：降低各行业接入 AI 3D 的门槛。
- **功能**：提供覆盖美妆、3C、鞋服等垂直行业的预置知识库（Expert Datasets）与提示词模板。

---

## 3. 核心突破：材质语义对齐 (Special Highlight)

> [!IMPORTANT]
> **“材质语义对齐”是 3.0 版本的核心技术竞争力。**
> 系统需建立从“用户感性语言”（如：丝滑、奢华金属、朦胧感）到“生成模型物理参数”（Roughness: 0.2, Metalness: 0.8, Refraction: 1.5）的精确映射矩阵。

---

## 4. 后期验收关键指标 (KPIs)

### 4.1 检索精度指标

| 指标                           | 目标值             | 测量方法                                     | 验收标准                    |
| ------------------------------ | ------------------ | -------------------------------------------- | --------------------------- |
| **Precision@3**                | 相比 v2.0 提升 20% | 建立 50+ 标准问答对，运行 `npm run test:rag` | v2.0 基准值 × 1.2           |
| **Precision@5**                | > 80%              | 同上，取 Top-5 结果                          | 5 条结果中 ≥4 条相关        |
| **Recall@10**                  | > 90%              | 人工标注相关性，计算召回率                   | 10 条结果覆盖 ≥90% 相关知识 |
| **MRR (Mean Reciprocal Rank)** | > 0.7              | 第一个相关结果排名倒数均值                   | 平均首位命中率 > 70%        |

**数据采集方式**：

- 构建标注数据集：`tests/rag/eval-dataset.json`（Question + Expected Knowledge IDs）
- 自动化测试脚本：`scripts/test-rag.ts`，输出 Precision/Recall/MRR
- 基准对照：v2.0 数据保留于 `docs/benchmarks/rag-v2.0-baseline.md`

### 4.2 Agent 工作流指标

| 指标               | 目标值 | 测量方法                       | 验收标准                 |
| ------------------ | ------ | ------------------------------ | ------------------------ |
| **一次性通过率**   | > 85%  | QC Agent 审核通过 / 总生成次数 | 首次提交无需重试         |
| **工具调用准确率** | > 95%  | 人工抽检 100 次 + 自动断言     | 参数正确 + 结果符合预期  |
| **端到端成功率**   | > 90%  | 完整工作流执行成功 / 总请求    | 从输入到 3D 模型导出成功 |
| **平均执行时长**   | < 60s  | Tracer 日志统计                | 用户输入 → 模型生成完成  |

**数据采集方式**：

- Agent Tracer：`lib/agent/tracer.ts` 记录每次执行的 Thought/Action/Observation
- 审核日志：QC Agent 决策记录存入 Supabase `agent_audit_logs` 表
- 面板展示：`/agent/metrics` 页面实时统计

### 4.3 材质语义对齐指标（核心突破）

| 指标               | 目标值   | 测量方法                        | 验收标准               |
| ------------------ | -------- | ------------------------------- | ---------------------- |
| **语义覆盖度**     | 50+ 材质 | 统计 `material_params` 知识条目 | 覆盖电商常见材质类别   |
| **参数推理准确率** | > 85%    | 人工对比描述 → PBR 参数映射     | 与专家标注差异 < 15%   |
| **生成质量评分**   | > 8/10   | 盲测：5 名评审打分              | PBR 参数应用后渲染效果 |

**数据采集方式**：

- 材质测试集：`tests/material/test-cases.json`（描述 + 期望参数）
- 推理 API：`/api/rag/material-params` 返回参数并记录命中率
- 质量评估：人工评审 + CLIP 相似度自动评估

### 4.4 商业化指标

| 指标                | 目标值 | 测量方法                    | 验收标准               |
| ------------------- | ------ | --------------------------- | ---------------------- |
| **生成→导出转化率** | > 30%  | 导出按钮点击 / 生成成功次数 | PostHog 事件追踪       |
| **用户留存率 (D7)** | > 25%  | 7 日后回访用户 / 首日用户   | Supabase Auth 用户分析 |
| **NPS (净推荐值)**  | > 30   | 用户满意度问卷              | 内置 NPS 弹窗收集      |

**数据采集方式**：

- 行为追踪：PostHog 集成，关键事件埋点（生成、导出、发布）
- 用户分析：Supabase Dashboard + 自定义 SQL 查询
- 满意度：生成完成后弹出 NPS 评分组件

### 4.5 性能与稳定性指标

| 指标                 | 目标值  | 测量方法              | 验收标准           |
| -------------------- | ------- | --------------------- | ------------------ |
| **RAG 检索延迟**     | < 500ms | API 响应时间 P95      | 包含 Rerank 全流程 |
| **Agent 执行超时率** | < 5%    | 超时失败 / 总执行次数 | 120s 超时阈值      |
| **API 错误率**       | < 1%    | 5xx 响应 / 总请求     | 健康检查告警       |

**数据采集方式**：

- APM 集成：Vercel Analytics + 自定义性能中间件
- 告警机制：错误率 > 1% 触发 Slack/邮件通知
- 日志聚合：结构化日志输出至 `.omc/logs/` 目录

## 5. 验收流程与里程碑

### 里程碑检查点

| 里程碑                    | 时间节点        | 验收内容                  | 验收方式                |
| ------------------------- | --------------- | ------------------------- | ----------------------- |
| **M1: RAG 基准建立**      | Phase 1 第 2 周 | 50+ 标准问答对 + 基准报告 | `npm run test:rag` 通过 |
| **M2: Agent Tracer 上线** | Phase 1 第 4 周 | 执行追踪 + 回放面板       | 演示完整工作流回放      |
| **M3: 材质知识库完成**    | Phase 1 结束    | 50+ 材质参数映射          | 推理准确率测试通过      |
| **M4: Multi-Agent 协作**  | Phase 2 中期    | 三角色协作工作流          | QC 拒绝率统计 + 演示    |
| **M5: 商业化闭环**        | Phase 3 结束    | 多平台发布 + 编辑器       | 端到端用户测试          |

### 持续监控机制

1. **每日构建**：CI 自动运行 RAG 测试 + Agent 冒烟测试
2. **周报生成**：自动统计 KPI 指标并生成 Markdown 报告
3. **月度评审**：对照 PRD 目标调整优先级和资源分配
