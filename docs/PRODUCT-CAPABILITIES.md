**版本**: v1.0  
**创建时间**: 2026-03-19  
**视角**: 营销 + 用户视角反推产品能力

## 📌 执行摘要

**核心原则**: 从用户场景反推功能，而不是从技术正推功能

**三大核心能力**:
1. **材质语义对齐** - 用户说"丝滑"，系统懂 Roughness:0.2
2. **一键发布小红书** - 生成→适配→文案→发布，全流程自动化
3. **电商场景优化** - 懂平台规范、懂电商语言、懂卖家需求

## 👥 一、用户场景地图

### 场景 1: 女装卖家上新

#### 用户故事
```
王小姐，28 岁，淘宝女装店主
周一上午 10 点，收到新款连衣裙样衣
需要：30 分钟内完成产品图，赶上下午 3 点上新

当前流程:
1. 自己用手机拍产品图（20 分钟）
2. 用美图秀秀修图（15 分钟）
3. 写商品描述（10 分钟）
4. 上传淘宝 + 发小红书预告（10 分钟）
总计：55 分钟

痛点:
- 拍出来的图不够高级，像地摊货
- 不会打光，产品质感差
- 小红书不会写文案，没人看
```

#### easy3d 理想流程
```
1. 上传连衣裙照片（手机拍摄即可）
2. 输入描述："温柔风连衣裙，适合春夏，想要高级感"
3. 系统自动:
   - 识别产品类别：连衣裙
   - 推荐材质：雪纺/棉麻（可调整）
   - 推荐场景：咖啡厅/户外花园
   - 生成 3D 展示（30 秒）
4. 一键发布:
   - 自动裁剪淘宝主图尺寸（800x800）
   - 自动裁剪小红书尺寸（3:4）
   - 自动生成文案："春夏温柔风连衣裙✨高级感拿捏了"
   - 自动添加标签：#女装 #连衣裙 #穿搭
5. 完成！

目标时长：5 分钟
```

### 能力 1: 材质语义对齐（核心壁垒）

#### 用户语言 → PBR 参数映射

**用户输入**:
```

"朦胧的磨砂感"
"科技感，冷色调"
```

**系统输出**:
```json

  "description": "丝滑",
  "params": {
    "roughness": 0.2,
    "metalness": 0.1,
    "clearcoat": 0.8,

  }
}
```

#### 技术实现
```typescript
// lib/rag/material-params.ts

interface MaterialParams {
  roughness: number;
  metalness: number;
  clearcoat?: number;
  anisotropy?: number;
  transmission?: number;
}

async function inferMaterialParams(description: string): Promise<MaterialParams> {
  // 1. 检索材质知识库
  const knowledge = await ragSearch(description, 'material_params');
  
  // 2. LLM 推理（处理模糊语义）
  const llmParams = await llm.infer(`
    用户描述：${description}
    参考知识库：${JSON.stringify(knowledge)}
    
    请输出 PBR 参数（roughness/metalness/clearcoat/anisotropy）
  `);
  
  // 3. 融合结果
  return mergeParams(knowledge, llmParams);
}
```

| 指标 | 目标值 | 测量方式 |
|------|--------|----------|

| **参数推理准确率** | >85% | 人工对比描述→参数 |
| **生成质量评分** | >8/10 | 盲测评审打分 |

#### 小红书 API 集成
```typescript
// lib/publish/xiaohongshu.ts

interface XiaohongshuPost {
  title: string;
  content: string;
  images: string[]; // 图片 URL 或文件路径
  tags: string[];
}

async function publishToXiaohongshu(post: XiaohongshuPost): Promise<{
  success: boolean;
  postId?: string;
  postUrl?: string;
  error?: string;
}> {
  // 1. OAuth 2.0 授权（用户首次使用）
  const token = await getAccessToken();
  
  // 2. 上传图片
  const imageIds = await uploadImages(post.images, token);
  
  // 3. 发布笔记
  const result = await fetch('https://api.xiaohongshu.com/note/publish', {
    method: 'POST',
    headers: { Authorization: `Bearer ${token}` },
    body: JSON.stringify({
      title: post.title,
      desc: post.content,
      image_ids: imageIds,
      topics: post.tags,
    }),
  });
  
  return result.json();
}
```

| 指标 | 目标值 | 测量方式 |
|------|--------|----------|

| **文案采纳率** | >80% | 用户未修改直接发布 |

#### 智能尺寸适配
```typescript
// lib/export/platform-adapter.ts

interface PlatformSpec {
  width: number;
  height: number;
  aspectRatio: string;
  maxSizeMb: number;
  format: string[];
}

async function adaptToPlatform(
  model: ThreeModel,
  platform: string
): Promise<ExportResult> {
  const spec = getPlatformSpec(platform);
  
  // 1. 调整相机视角
  camera.aspect = spec.width / spec.height;
  camera.updateProjectionMatrix();
  
  // 2. 调整灯光（不同平台光线要求不同）
  setupLightingForPlatform(platform);
  
  // 3. 渲染输出
  const image = await render(model, camera, spec);
  
  // 4. 压缩优化
  const optimized = await optimizeImage(image, spec.maxSizeMb);
  
  return optimized;
}
```

## 💬 小 Q 的建议 🌀

### 从用户视角看产品能力

**你现在的技术能力**:
- ✅ RAG 检索（但效果差，Precision@5 仅 20.97%）
- ✅ Agent 工作流（单 Agent，无 Multi-Agent）
- ✅ 3D 生成（Tripo API 集成）

**用户需要的能力**:
- ❌ **材质语义对齐** - 用户说"丝滑"，系统要懂
- ❌ **多平台适配** - 淘宝/小红书尺寸不同
- ❌ **文案生成** - 用户不会写小红书文案
- ❌ **批量处理** - 卖家一次上新 10 个产品

**建议优先级**:
```
第 1 周：材质语义对齐（50+ 材质库）
第 2 周：多平台尺寸适配 + 文案生成
第 3 周：RAG 优化（混合检索 + 查询改写）
第 4 周：批量生成 + 用户测试
```

**记住**：
- 用户不为"混合检索"买单，为"生成效果好"买单
- 用户不为"Multi-Agent"买单，为"30 分钟搞定上新"买单
- 从用户价值反推技术优先级，不要从技术正推
